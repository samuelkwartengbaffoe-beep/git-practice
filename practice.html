Day 1 – Git Basics & Local Version Control

Topics Covered

  • git init – start a new Git repository
  • git status – check changes in your working directory
  • git add – stage changes
  • git commit – save snapshots of your project
  • git log – view commit history
  • Writing good commit messages

Why These Commands Matter

These commands form the foundation of using Git. They help you track changes, organize your work, and save different versions of your project. Learning them is the first step toward collaborating on projects and working with remote repositories like GitHub.

Lab Activity

Create a new folder, initialize a Git repository inside it, and make 3 meaningful commits:

  • Commit 1: Add an HTML file
  • Commit 2: Add styling or another file
  • Commit 3: Edit your file and commit again

Challenge

Undo the last commit without losing your changes using:

git reset --soft HEAD~1

This removes the commit but keeps all your changes in the staging area.

New Concepts Added

  • git branch – create and manage branches
  • git switch – move between branches
  • git merge – combine branches
  • .gitignore – ignore unnecessary files

Branching Example

Extra Notes

Branches help keep your work organized by allowing you to create separate lines of development. Merging lets you bring changes together when ready. Using .gitignore keeps your repository clean and prevents committing system files or secrets.
